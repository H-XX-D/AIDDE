# Contributing

AIDDE is early. Contributions should help clarify or build the rewrite, not add
surface area for its own sake.

Good contributions:

- improve the panel primitive,
- clarify agent observability,
- improve memory/preflight UX,
- make audit data more useful,
- strengthen learning workflows,
- improve DD state communication,
- document real beta feedback.

Avoid:

- dumping unrelated prototype files,
- adding broad integrations before core workflow is stable,
- hiding safety state behind personality,
- storing secrets or private data in examples,
- turning the audit log into chat transcript storage.

## Development stage

Implementation guidelines will become stricter once the rewrite source lands.
For now, design docs and small scaffolding contributions are welcome.
