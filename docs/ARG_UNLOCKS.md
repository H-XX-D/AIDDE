# Swag and ARG Unlocks

AIDDE's reward layer should make usage, learning, and discovery feel tangible.

## Unlock types

1. Explicit achievements
   - complete a learning mission,
   - ship a project,
   - run tests successfully,
   - recover from a broken state,
   - use memory preflight correctly,
   - promote an audit trail into a macro.

2. Mastery achievements
   - use panel targeting,
   - keep risky files out of scope,
   - catch a bad memory preflight,
   - run a clean review loop,
   - resolve a hard bug with tests.

3. Hidden ARG-style unlocks
   - time-window actions,
   - seasonal actions,
   - specific command sequences,
   - weird but harmless rituals,
   - rare project milestones,
   - long streaks,
   - discovery through curiosity.

4. Physical unlocks
   - STL exports,
   - printable DD accessories,
   - printable badges,
   - printable desk objects tied to achievements.

## Design rule

Hidden unlocks should delight users, not block functionality.

The core IDE should remain clear. The hidden layer should reward exploration,
not make normal work confusing.

## Why it matters

Most development tools treat personality as decoration. AIDDE can make
personality part of retention, learning, and product identity.

DD's wardrobe, unlocks, and printable accessories connect software use to
something visible and collectible. That is a real product advantage if it stays
honest and does not get in the way.
