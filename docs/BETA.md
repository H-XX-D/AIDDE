# Beta Status

AIDDE is in beta-stage design and rewrite preparation.

## What beta means here

Beta does not mean "the idea is vague." The direction is specific:

- agent-native panels,
- DD sidecar and title-bar identity,
- memory preflight,
- Recall/MAL integration,
- action audit,
- learning system,
- reward and unlock layer,
- future 3D printable DD accessories.

Beta means the public rewrite is being assembled from lessons learned in the
working AIDE prototype.

## Current prototype evidence

The private/prototype AIDE work has already explored:

- DD mascot and sidecar behavior,
- Electron-based desktop shell,
- panel switching,
- editor, explorer, terminal, console, chat, Recall, and audit surfaces,
- Recall memory-in-use display,
- command/preflight ideas,
- action auditing,
- layout persistence and panel rewrite planning.

## What should not happen in beta

- Do not expose private prototype debris just to make the repo look full.
- Do not over-integrate every adjacent tool before the shell is stable.
- Do not make gamification interfere with safety or shipping.
- Do not let chat prose become the audit log.
- Do not make memory invisible.

## What should happen first

The first useful beta should prove:

- the panel primitive works,
- the command rail feels natural,
- memory preflight can be stopped and edited,
- agent actions are visible in Audit,
- DD state is clear,
- the learning start page has a real point of view.
