# Learning System

AIDDE's learning wedge is not generic coding tutorials. The wedge is learning
how to build with AI.

## Learning goals

AIDDE should help users learn:

- how to ask for useful work,
- how to keep scope tight,
- how to spot bad context before a turn starts,
- how to read diffs,
- how to use tests,
- how to recover from agent mistakes,
- how to review model actions,
- how to turn repeated workflows into macros,
- how to ship without depending on blind trust.

## Product surfaces

The learning system should appear in:

- the start page,
- project templates,
- tutorials,
- kneeboard guidance,
- DD achievements,
- unlockable cosmetics,
- compact hints attached to real work.

## What makes it different

The goal is not to pause work and send the user to a course.

The goal is to teach inside the workflow:

- while a prompt is being formed,
- while memory is being selected,
- while an agent is editing,
- while a test fails,
- while a risky diff appears,
- while the user promotes an audit trail into a macro.

Good learning in AIDDE should feel like better tools, not homework.
